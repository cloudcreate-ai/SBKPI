## 2. 游戏定位（主线 A）

- **模式**：**PVE、1v1**（玩家 vs 关卡敌人）；可扩展 PVP / 2v2，但讨论焦点先在1v1。
- **核心体验**：围绕**职场任务**与 **KPI 推进**，不是传统「把对方生命值打到 0」唯一胜负（尽管早期草稿出现过能量+攻防牌思路）。
- **胜负核心**：在**回合时限**内达成任务指标，并满足**风险 / 质量**等约束；失败含「截止爆雷」「压力/风险爆表」等语义包装。
- **参考手感（讨论用语）**：
  - **类三国杀**：节奏上「进度 vs 风险」拉扯、领导技能施压。
  - **类小丑牌**：**花色 / 连打 / 条件联动**，用连携处理「需求变更」类规则。

## 3. 人格与数值如何接入

- **SBTI 15 维**保留：可映射为**五组**（自我 / 情感 / 态度 / 行动 / 社交），维上 L/M/H 对应数值（如 1/2/3），再归一到 **Attack、Guard、HealPower、ControlPower、Tempo** 等战斗向属性（具体公式在对话中有草稿，此处不锁死）。
- **人格战斗分类（讨论稿）**：主攻 / 主防 / 均衡 / 奶妈 / 爆发；每类有推荐系数模板，便于批量填表。
- **技能设计原则**：须能对应 **文案关键词 + 维度高低 + 可感知战斗行为**（控场、爆发、续航、资源），避免换皮同质化。
- **特殊结果**：**DRUNK**、**HHHH** 在对话中有多种定位（状态触发、高随机保底、异常流派等），**未最终定稿**。

## 4. 任务与 KPI 模型（讨论稿）

- **多指标**（示例命名）：  
  - **里程碑完成度 / KPI**（主进度）  
  - **项目风险热度 / Pressure**（过高失败）  
  - **D-Day / 剩余回合**  
  - **可交付可信度 / Quality**（过低惩罚或返工）
- **任务类型举例**：冲刺型（抢进度）、稳交付型（均衡）、审计型（质量与高压）。
- **Roguelike 壳（讨论）**：过关后从随机人格中**招募同事**；下一关难度与任务类型变化；**领导人格**可改关卡规则（与「老板」主题绑定）。
- **星级（讨论）**：按提前量、风险、质量等给星，影响后续招募池质量。

## 5. 工作流与牌面（R → D → C → T）

- 工作流由「开会」收敛为更清晰四段：  
**R需求讨论 → D 文档 → C 开发 → T 测试发布**。
- **流程感为核心**：一回合内尽量打出**完整链**；缺环节用**软惩罚**（倍率下降），而非直接判负。
- **扑克化讨论**：牌可带 **点数 + 花色**；点数贡献基础产能，**完整顺序链**触发 **Flow 倍率**，**同花/跨花色**可有 **Suit 倍率**；**加班、救火、需求变更**等作**修正牌**，通常带代价（如加压）。
- **防卡手（讨论列举）**：起手保底、缺失池发现、通配、检索、连败补偿、构筑下限等——目标是从「纯抽脸」变成「可规划」。

---

## 6. 关键共识：四普通槽 + 一增益槽

讨论中**明确认可**的结构（为降低断流程的随机性）：

- **4 个普通槽**：对应 **R / D / C / T**，每槽有上限（如 4），**每回合至少补 1**，避免某阶段永久断档。
- **1 个增益槽**：放**加班、救火、协调、需求变更**等修正牌，上限与补牌概率单独设（如上限 3、概率补牌）。
- **出牌**：每槽每回合可出 **1～N 张**（讨论建议 N 从 2 试起）；增益牌每回合限制张数，避免一回合叠爆。
- **结算**：四槽都出过牌视为「完整流程」，给较高 **FlowMul**；缺槽则降档；**同槽多张**建议**边际递减**；**同类增益**建议**同回合只生效一次**等防失衡条。

具体数值（槽上限、补牌数、倍率表）在对话中仍为**待定**。

## 7. 场景样例（讨论用，非定稿）

- **「周一例会保卫战」**：对领导 **CTRL**，限时内 KPI 进度达标且风险不爆；领导技能偏**加费、定时加压**；玩家用**同步进度 / 加班 / 复盘**等牌在进度与风险间取舍。  
- **「需求评审拉扯局」**：对领导 **OJBK**，多子 KPI + **需求变更**未处理则衰减；**THIN-K** 例强调**花色连打、冻结变更**等「小丑牌感」联动。

用于验证**机制趣味**，不是最终关卡列表。

## 9. 收敛：54 张人格扑克（主线 B，已落地）

讨论后期与当前仓库一致的方向：

- **不引入性别维度**；以 **27 人格**为核，每人格 **领导 / 同事** 两套语境（黑话 + 技能一条）。
- **54 张**对应标准扑克 + **Joker**（如 DRUNK 双张）；红/黑或 Boss/Coworker 与牌组颜色、角色字段对应。
- 技能语气：**搞笑黑话**，但效果描述保持**可算、可执行语感**，便于日后若接战斗系统。
- **数据落盘**：`[persona-poker-deck.csv](./persona-poker-deck.csv)`（字段以表头为准）。
